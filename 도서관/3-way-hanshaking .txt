# 3-way-hanshaking 

- 개념 
(1) tcp 프로토콜 시, 데이터의 전달 전에 통신을 위한 연결을 위해 시행
(2) 정확한 전송을 위해 상대 컴퓨터와 사전에 세션을 수립하는 과정

- 과정
(1) 클라이언트 => 서버 : 접속을 요청하는 syn 패킷을 보내고, sys-sent 상태로 대기

(2) 서버 => 클라이언트 : syn 요청을 받고, 클라이언트에게 요청을 수락한다는
ack, sysnflag가 설정된 패킷을 발성, ack로 응답하기를 기다리는 syn_received 상태가 된다

(3) 클라리언트 => 서버 : ack을 보내고 이후부터는 연결
