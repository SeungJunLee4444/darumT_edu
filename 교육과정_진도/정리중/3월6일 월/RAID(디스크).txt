## RAID

- 개념 : 여러 디스크를 배열하여 속도, 안정성, 효율성, 가용성을 증대시키는 기술 *
- 장점
(1) 데이터 안정성 증대
(2) 디스크 용량 증설에 용이
(3) 디스크 입출력 성능 향상

- 종류
(1) RAID 0
	- 두개 이상의 하드디스크 필요
	- 모든 디스크에 동시에 저장
	- 장점 : 성능이 빠름, 100퍼센트의 공간효율 ex) 1gb + 1gb = 2gb
	- 단점 : 디스크 장애시 복구가 어려움

(2) RAID 1
	- 동일한 용량의 두개의 디스크 필요
	- 장점 : 미러링, 같은 데이터를 중복하여 보존, 복원에 용이
	- 단점 : 공간효율이 나쁨 ex) 1gb + 1gb = 1gb

(3) RAID 5
	- 최소 3개 이상의 하드디스크로 구성, 패리티를 이용한 데이터 복원
	- 장점 : 1의 데이터 안정성과 0의 공간효율성을 가짐, 패리티를 이용한 데이터 복구
	- 단점 
		(1) 2개 이상의 하드디스크 장애시 데이터복구 불가능
		(2) 패리티 연산에 cpu에 부하

	* 패리티 : hhd에 장애가 발생할 때 데이터 복원을 위해 사용되는 부호로,
		raid에 데이터를 쓸 때, 자동적으로 생성된다
		ex 하드디스크가

(4) RAID 6
	- 최소 4개의 하드디스크 사용, 패리티를 이용한 데이터 복원
	- 장점 : 두개의 디스크에 장애가 발생해도 데이터 이상없음
	- 단점 : 5보다 신뢰도가 높아지지만, 공간효율이 떨어진다
