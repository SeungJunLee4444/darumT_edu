# 만들어진 그룹 확인

# 그룹생성
	- groupadd 그룹명			: 가장 마지막 그룹번호 다음번호를 자동으로 부여
	(1) groupadd -g gid(그룹번호) 그룹명	: 그룹번호를 임의로 부여
	* cat /etc/group로 그룹생성 여부 확인가능

# 계정생성
	- useradd 계정명 			: 가장 마지막 유저번호 다음번호를 자동으로 부여
	(1) -g	: 그룹 지정(이미 만들어진)
	(2) -u	: 유저번호 설정
	(3) -d	: 홈 디렉터리 설정 (계정명과 동일하게 만드는 것이 관례 *)
	(4) -s	: 쉘 설정
	(5) -p	: 비밀번호 추가
	* cat /etc/passwd로 계정생성 여부 확인가능
	
	* passwd 구조 설명
	 ex) etst555:x:2000:2002::/test555:/bin/bash
	  '계졍명 : 권한 : uid : gid :: /홈디렉터리 : 쉘'

# 파일에 대한 권한부여
	- 구조 : 사용자 권한 / 그룹권한 / other권한
	 ex) -rw-rw-r--: 사용자는 읽기쓰기, 그룹은 읽기쓰기, other은 읽기만가능
	- 권한 부여하기 : 문자 또는 숫자로 부여 가능하다
		chmod + 권한대상 + 권한
			u(사용자)	r(읽기)	4
			g(그룹)	w(쓰기)	2
			o(other)   	x(실행),	1
	* ls - al를 통해 권한 조회가능
	* 권한을 부여할려면 권한이 있는 계정만 가능하다 *매우중요 ex) root *

	ex) 일반적인 권한부여 형태	: chmod o+x	: other에게 실행권한 부여
	ex) 숫자로 권한 부여		: chmod 766 ./*	: 모든 파일에 사용자 rwd, 그룹 rw, other에 rw권한 부여

# 파일을 실행시킬 때, 실행권한이 없을 경우 대처법 2가지
	(1) chmod를 통해 소유주, 또는 root계정으로 파일에 대한 실행권한 부여
	(2) sh(쉘 명령어)를 통해 -x(실행권한제거) 옵션을 추가하여 실행
