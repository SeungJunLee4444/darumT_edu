+ vmware workstation pro 다운
- 용도 : 윈도우 또는 리눅스에서 여러 운용체제를 가상머신으로 실행하기 위한 소프트웨어

2. 리눅스
1) 특징 
- 명령어 기반 
- 다수의 사용자가 네트워크를 통해 한 시스템에 접근 가능
- 리눅스의 일종인 rethat enterprise linux는 lbm에서 구매

2) 구조 : 하드웨어, 커널, 쉘, application으로 구성
- 커널 : 리눅스에서는 운영체제(os)를 커널이라 부르며, 하드웨어를 관리한다
- shell : 명령어 해석기, os 바깥부분에 위치하여 명령어를 해석

3) 프로세스 : 서버의 cpu에서 실행되는 프로그램을 말한다
- 구성
(1) pid : 실행될떄 각 프로세스의 고유번호가 부여되며 실행(pid)
(2) ppid * : 부모프로세스의 id, 숫자가 같으면 부모, 자식 프로세스 관계를 의미한다
* 부모 pid를 종료하면, 종속된 모든 ppid관계의 프로세스가 종료된다
* 부모 프로세스를 죽여도 살아있는 좀비프로세스는 자원을 계속 소모하기 떄문에, 별도의 처리가 필요하다

- 명령어 : ps -ef

4) 디렉터리 구조
- 특징 
(1) 디렉터리(폴더) 파일을 관리하기 위해 트리로 구조되있음
(2) 최상위 계정은 'root' 계정

- 디렉토리
(1) home : 홈 디렉토리가 생성되는 곳
(2) media : 
(3) opt : 소프트웨어가 설치되는 디렉토리
(4) dev : 장치파일이 저장되있는 디렉터리
(5) root : root계정의 홈 디렉터리(root 디렉터리x)
(6) sys	: 리눅스 커널 관련 정보가 담긴 디렉터리
(7) bin : 기본 명령어가 모여있는 디렉터리 / sbin : 슈퍼명령어 모음(root계정용)
(8) var : 시스템 운영중에 사용되는 로그가 저장되는 디렉터리
(9) tmp : 임시데이터 저장, 부팅시 초기화됨 * 중요한 파일을 저장하면 안됨
(10) mmt : media와 비슷한 역할, 파일시스템 연결
(11) etc : 리눅스 설정을 위한 파일을 가진 디렉터리(ex 포트번호)


5) 부팅과정( 전원 -> os)
(1) os가 디스크에 저장되있음
(2) os를 실행시키려면 메모리에 저장시켜야함(ram, rom)
(3) cpu가 rom의 부트로더에 명령을 내려서 os를 실행시킴

5. 부팅과정
- 정의 : 전원을 킬 때, 운영체제를 메모리에 올리는 과정
- 순서
(1) cpu가 제일 먼저 부팅된다
(2) cpu는 메모리의 rom(비휘발성 메모리)에 저장된 bios(basic input output system) 데이터를 읽는다
(3) bios는 post(power on selft text)를 진행시켜 하드웨어의 정상작동을 검사
(4) 이상이 없으면, bios는 disk의 mbr에 저장된 부팅정보를 ram(메모리)로 읽어온다
(5) 부트로더(운영체제를 메모리로 읽어오는 역할) disk에 있는 os 코드를 메모리로 읽어온다
(6) cpu는 읽어온 os명령을 첫 프로세스로 실행한다
