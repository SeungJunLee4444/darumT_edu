## OSI 7계층

- 모든 통신은 tcp/ip와 osi 7 layer라는통신표준에 따라 설계되어있다

* 네트워크 내에서 프로토콜을 통해 정해진 규약에 따라 통신하는데 이를 프로토콜이라한다


- 구분이유 : 통신이 일어나는 과정을 파악하여 네트워크 오류 원인을 파악

1. 구조
	
application layer	(7) 응용계층 : 사용자와 가장 밀접한 계층, 인터페이스
	|	- 예시 : 소프트웨어(컴퓨터)
	|
session layer	(6) 표현계층 : 데이터를 어떻게 표현할지 역할을 정하는 계층
	|	- 송신자에서 온 데이터 해석을 위한 응용계층 데이터 부호화
	|	- 데이터의 암호화 복호화
	|	
	|
transport layer	(4) 전송계층 : 통신 활성화를 위한 계층, 주로 tcp 프로토콜 사용 
	|	- 기능
	|		패킷전송의 유효여부를 확인하고 전송실패한 패킷을 다시 전송
	|	- 특징 
	|		(1) 전송을 위해 port 번호를 사용
	|		(2) 데이터 수신시, 4계층에서 해당 데이터를 하나로 합쳐서 5계층에 던져준다
	|	* 프로토콜의 종류
	|		(1) TCP	: 서버와 클라이언트 간에 데이터를 신뢰성있게 전달하기 위한 프로토콜
	|		- 특징
	|			(1) 연결지향성, 신뢰성 높음
	|			(2) 3-way handshaking을 통해 연결 후 통신시작
	|		(2) UDP	: 비연결성 프로토콜, 데이터를 주고받기 전에 연결 절차를 거치지않고 발신
	|		- 특징 : tcp보다는 빠른 전송이 가능하지만 데이터 신뢰도가 낮음
	|
networks layer	(3) 네트워크 계층 
	|	- 기능 : 경로를 선택하여 패킷을 전달하는 계층, 즉 라우팅을 하는 계층이다
	|	- 장비 : 라우터(공유기), L3스위치
	|	- 확인법 : cmd - ipconfig/all 명령어로 조회가능
	|
data link layer	(2) 데이터 링크 계층
	|	- 기능 : 안전하게 데이터가 전송되도록 돕는 역할을 하는 계층
	|		(1) 통신 오류
	|		(2) 재전송 기능	
	|	- 특징 : 맥 주소(mac adress)를 사용하여 통신한다
	|	- 장비 : 브리지, 스위치(L2)		
	|
physical layer	(1) 물리적계층
		- 기능 : 오직 데이터 전달만을 담당(데이터가 무엇인지, 어떤 에러가 있는지 신경x)
		
		- 장비 : 케이블, 리피터, 허브 등





L1 케이블, L2 탭, 케이블연결, L3 라우팅 스위치, L4 프로토콜의 header을 분석하여 서버 부하를 분산
